## 1. bcrypt.hash 加密密碼

```js
const bcrypt = require('bcrypt');

const hash = await bcrypt.hash(password, 12); // 12 = salt rounds
```

- 單向加密，把明碼變成 `$2b$...` 的 hash 存起來，**明碼絕不進資料庫**。
- 是 async，一定要 `await`，所以 handler 要寫成 `async function`。

> ⚠️ 也可以先 `genSalt` 再 `hash`，但直接 `bcrypt.hash(明碼, rounds)` 一步到位最常用。

---

## 2. bcrypt.compare 驗證密碼

```js
const ok = await bcrypt.compare(password, user.password);
// ok 是 boolean：明碼與 hash 相符 → true
```

- 比對「使用者輸入的明碼」與「資料庫存的 hash」。
- 同樣是 async，忘記 `await` 會拿到 Promise（永遠為 truthy）而判斷錯誤。

---

## 3. jwt.sign 簽發 token

```js
const jwt = require('jsonwebtoken');

const payload = { id: user.id, email: user.email };
const token = jwt.sign(payload, process.env.JWT_SECRET, { expiresIn: '30d' });
```

- 三個參數：`jwt.sign(payload, secret, options)`。
- `iat`（簽發時間）和 `exp`（過期時間）由套件**自動加上**，不用自己寫；`expiresIn` 會被換算成 `exp`。

> ⚠️ `expiresIn` 要放進第三個參數的 **options 物件**（`{ expiresIn }`），不是獨立參數。

---

## 4. jwt.verify 驗證 token

```js
try {
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.user = decoded; // decoded 就是當初簽的 payload + iat/exp
  next();
} catch (error) {
  return res.status(401).json({ status: 'false', message: 'Token 無效或已過期' });
}
```

- 成功回傳 decoded payload；**失敗會 throw**，所以要包在 `try/catch`。
- 驗過要**手動**把 decoded 掛到 `req.user`，並呼叫 `next()`。

## 5. 解析 Authorization header（Bearer token）

```js
const authHeader = req.headers.authorization; // 'Bearer eyJhbGci...'
if (!authHeader || !authHeader.startsWith('Bearer ')) {
  return res.status(401).json({ status: 'false', message: '請先登入' });
}
const token = authHeader.split(' ')[1]; // 取 Bearer 後面那段
```

- 格式固定是 `'Bearer <token>'`，中間一個半形空格。

> ⚠️ 是 `'Bearer '`（含結尾空格），不是 `'Bear '`——少字母會讓所有正確 token 都被擋。

## 6. Express middleware 與 next()

```js
// 一般 middleware：3 個參數
const mw = (req, res, next) => {
  // 做事…
  next(); // 放行給下一個
};

// 掛在單一路由前當守門員
router.get('/me', verifyToken, (req, res) => { /* ... */ });
```

- middleware 靠 `next()` 把控制權交給下一站；不呼叫 `next()` 且不回應，請求就會卡住。

---

## 7. Express 錯誤處理 middleware（4 參數）

```js
app.use((err, req, res, next) => {
  res.status(500).json({ err: err.name, message: err.message });
});
```

- **一定要 4 個參數**（`err` 在最前面），Express 才認得它是 error handler。
- 放在所有路由**最後面**。`express.json()` 遇到壞掉的 JSON 會 throw `SyntaxError`，自動被它接住。

---

## 8. app 掛載順序

```js
app.use(cors());                                   // 1. 跨域
app.use(express.json());                            // 2. 解析 JSON body
app.use('/docs', swaggerUi.serve, swaggerUi.setup(swaggerDoc)); // 3. Swagger
app.use('/auth', authRouter);                       // 4. 業務路由
app.use((req, res) => res.status(404).json({ /* ... */ })); // 5. 404
app.use((err, req, res, next) => { /* ... */ });    // 6. error handler（最後）
```

- 順序即執行順序：body 解析要在路由前；404 要在所有路由後；error handler 永遠最後。

---

## 9. 陣列查找：find 與 some

```js
const user = users.find(u => u.email === email); // 回傳「元素」或 undefined
const exists = users.some(u => u.email === email); // 回傳 boolean：有沒有符合的
```

- 找「是誰」用 `find`；只問「存不存在」用 `some`。
