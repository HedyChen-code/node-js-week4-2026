## 錯誤總表

| # | 坑 | 錯誤寫法 | 正確寫法 |
|---|---|---|---|
| 1 | `jwt.sign` 參數放錯 | `jwt.sign(payload, secret, iat, expiresIn)` | `jwt.sign(payload, secret, { expiresIn })` |
| 2 | async 忘了加 | `router.post('/login', (req, res) =>` | `router.post('/login', async (req, res) =>` |
| 3 | 註冊判斷式邏輯反了 | `!users.some(u => u.email === email)` | `users.some(u => u.email === email)` |
| 4 | Bearer 少字母 | `authHeader.startsWith('Bear ')` | `authHeader.startsWith('Bearer ')` |
| 5 | `iat` 當成裸變數 | `iat,` | `iat: req.user.iat` |
| 6 | token 過期欄位取錯 | `exp: req.user.expiresIn` | `exp: req.user.exp` |
| 7 | `nextId++` 位置害 id 跳號 | 先 `nextId++` 再拿來當 id | `id: nextId++` |
| 8 | 方法／屬性打錯字 | `.josn`、`teq.body`、`.status({...})`、`toekn` | `.json`、`req.body`、`.json({...})`、`token` |
| 9 | `gh auth switch` 用錯參數 | `gh auth switch HedyChen-code` | `gh auth switch --user HedyChen-code` |

## 逐項說明

### 1. `jwt.sign` 的第三個參數是 options 物件

`jwt.sign` 只有三個參數：`(payload, secret, options)`。`expiresIn` 要放進 options：`{ expiresIn: '30d' }`。而 `iat`、`exp` 是套件**自動**加進 token 的，根本不用你寫。當初把 `iat, expiresIn` 直接當參數傳，是把「token 內容」和「函式參數」搞混了。

### 2. 用了 await 就一定要 async

`register` 和 `login` 裡都有 `await bcrypt.xxx`，但 handler 忘了寫 `async`，會直接語法錯誤。**看到 `await` 就回頭確認外層函式是不是 `async`**。

### 3. `some` 前面不該加 `!`

`users.some(u => u.email === email)` 回傳的是「email **是否已存在**」。email 已存在時要擋（回 400），所以條件就是 `some(...)` 為 true。加了 `!` 變成「不存在時才擋」，邏輯整個反過來——正常註冊被擋、重複 email 反而放行。

### 4. `'Bearer '` 少一個 er 全盤皆錯

`startsWith('Bear ')` 少了 `er`，會讓所有正確的 `Bearer <token>` 都被判定格式錯誤，導致「帶有效 token」的測試也失敗。字串比對這種地方要一字不差。

### 5 & 6. `req.user` 是 decoded payload，欄位要對

`/me` 裡要回傳的 `iat`、`exp` 都在 `req.user`（也就是 `jwt.verify` 解出來的 payload）裡：
- `iat,` 這樣寫是「引用一個叫 iat 的變數」，但根本沒宣告 → ReferenceError，整支 API 500。要寫 `iat: req.user.iat`。
- token 的過期欄位叫 `exp`，不是 `expiresIn`（`expiresIn` 只是簽發時給 `jwt.sign` 的參數名，不會進 payload）。

### 7. `id: nextId++` 一步到位

`nextId` 起始是 2（管理員佔 id 1）。先寫 `nextId++` 再拿 `nextId` 當 id，第一個新用戶會變成 3、跳過 2。用 `id: nextId++`（先取值再遞增）就對了。這次測試沒驗 id 所以沒紅，但邏輯是錯的。

### 8. 純打錯字（沒有觀念問題，就是累）

`.josn`→`.json`、`teq.body`→`req.body`、`res.status(200).status({...})`→`.json({...})`、`toekn`→`token`。這類錯誤 IDE 通常不會即時報，但一跑就爆。**寫完趁手感還在，快速自己讀一遍**比事後 debug 快。

### 9. `gh auth switch` 用旗標不用位置參數

這指令收 0 個位置參數，要用 `--user`（或 `-u`）指定帳號：`gh auth switch --user HedyChen-code`。注意 `--user` 要連在一起，`-- user`（中間有空格）會被當成「旗標結束」，反而報 `received 2`。只有兩個帳號時，直接 `gh auth switch` 跳互動選單最不會錯。

## 給自己的提醒

> - 看到 `await` 先確認 `async`。
> - 判斷式（尤其加不加 `!`）寫完，念一遍「這條成立時我要做什麼」。
> - 累了寫的 code，跑測試前先自己讀過一遍抓錯字。
